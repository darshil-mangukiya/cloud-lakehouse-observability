# Sample Outputs

This folder contains curated examples that help a GitHub reviewer understand the platform without tracking repeated runtime artifacts.

| File | What It Proves |
|---|---|
| `sample_release_gate_decision.md` | Gold datasets are checked before BI publication |
| `sample_schema_drift_event.md` | Source schema changes are detected and classified |
| `sample_data_quality_report.md` | Validation failures are captured with actionable context |
| `sample_dataset_readiness_report.md` | Freshness, quality, and readiness are summarized for gold data |
| `sample_incident_summary.md` | Alerts can be grouped into an operational incident summary |
| `sample_gold_mart_preview.csv` | Gold marts are shaped for analyst and BI consumption |
| `sample_lineage_graph.md` | Source-to-gold lineage is documented for impact analysis |

Generated pipeline artifacts, alert logs, scorecards, and dashboard exports are intentionally ignored by Git.
