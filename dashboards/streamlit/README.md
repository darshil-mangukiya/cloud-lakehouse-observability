# Streamlit Observability Dashboard

This local dashboard is the command center for the Cloud Data Lakehouse & Data Observability Platform. It reads generated repository artifacts and shows whether lakehouse data is reliable, observable, governed, and safe for BI consumption.

## Pages

| Page | What It Shows |
|---|---|
| Data Health | dataset readiness, source freshness, trust scores, release status, recent alerts |
| Data Quality | validation results, failed checks, rejected records, row-count anomalies |
| Schema Drift | schema registry history, contract outcomes, safe renames, breaking changes |
| Release Gates | BI publication decision, blocking reasons, trust/readiness status |
| Alerts And Incidents | alert severity, business impact, recommended action, incident summary |
| Gold Marts | gold datasets, fact/dimension views, row counts, mart purpose, preview tables |
| Metadata And Lineage | catalog ownership, criticality, downstream usage, lineage edges |

## Required Artifacts

The dashboard reads local artifacts generated by the platform pipeline:

- `observability/reports/*.json`
- `operations/release/release_gate_report.json`
- `alerts/alert_log.jsonl`
- `metadata/schema_registry.json`
- `metadata/contract_reports/**/*.json`
- `metadata/data_catalog.json`
- `lineage/lineage_graph.json`
- `data_quality/reports/*.json`
- `data_quality/reports/failed_records/*.jsonl`
- `gold_layer/*.csv`
- `dashboards/exports/*.csv`

Missing optional artifacts are shown as clean empty states with the message:

```text
Run make validate-platform to generate this artifact.
```

## Generate Artifacts

```bash
make validate-platform
```

## Run Dashboard

```bash
streamlit run dashboards/streamlit/app.py
```

The legacy entrypoint still works:

```bash
streamlit run dashboards/streamlit_app.py
```
